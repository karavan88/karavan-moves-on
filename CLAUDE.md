# CLAUDE.md — «Караван идёт»

Авторский сайт о кино Карена Аванесяна. Этот файл — источник правды по
**единым стандартам контента и оформления**. Любая сессия, редактирующая
репозиторий, обязана им следовать, чтобы страницы оставались консистентными.

## Архитектура (коротко)
- Статический сайт. SPA-оболочка `index.html` (весь CSS и роутер тут) +
  общий модуль рендера `assets/render.js` (используется И сборщиком, И
  браузером — поэтому функции в нём чистые, без обращения к DOM на верхнем уровне).
- Сборка: `npm run build` → `build.mjs` пререндерит каждую страницу в `dist/`
  (плюс sitemap, rss, robots). `dist/` в `.gitignore` — **не коммитим**.
- Деплой: Netlify собирает из `main` → https://karavancinema.netlify.app
- Контент: markdown + JSON-манифесты:
  `reviews/` (рецензии), `collections/` (подборки), `feed/` (заметки),
  `festivals/`, `press.json`. Картинки берутся с TMDB и «запекаются» локально
  (`tools/`, токен в `.env`).
- Темы: **тёмная — по умолчанию** (`:root`), светлая — `:root[data-theme="light"]`.
  Переключатель «солнце/луна» в шапке, выбор в localStorage. Скролл-истории
  следуют теме.

## Рабочий процесс
- После правок контента/рендера: `npm run build`, проверить, закоммитить
  **только изменённые файлы** (часто есть параллельная работа в `lectures/` от
  другой сессии — её не трогаем), затем `git push` (Netlify задеплоит).
- Коммитить/пушить только по явной просьбе. Сообщения — на русском, в стиле
  существующей истории.

## СТАНДАРТ КОНТЕНТА

### 1. Оценки фильмов
- **Единая шкала: 0–10** (допустимы половинки: 7.5). Хранится как `rating: 8`.
- **Три формы показа, и только они:**
  - бейдж на постере (карточки, featured): `★ N` — `ratingHTML(rating)` в render.js.
  - страница рецензии: ряд из 10 звёзд + `N / 10` — `ratingHTML(rating,'full')`.
  - инлайн в подборках/заметках: `<span class="rate">★ N/10</span>` (в строке метаданных).
- **Запрещено:** рисовать звёзды вручную с инлайн-цветом, использовать шкалу /5.
  Звёзды/`.rate` — латунные (`--gold`), пустые — `--gold-empty`, тема — автоматически.

### 2. Названия фильмов
Три поля, у каждого одна роль:
- `film` — **русское прокатное название в «ёлочках»**, и больше ничего.
  Без скобок, без оригинала внутри. ✅ `«Сообщите матери»` ❌ `«Сообщите матери» (Amma Ariyan)`
- `original` — оригинальное название. Порядок: **родное письмо → латиница/англ.**
  через ` / `. Для русских/советских фильмов — опускаем.
  ✅ `姿三四郎 / Sanshiro Sugata` · `어쩔수가없다 / No Other Choice`
- `title` (только рецензии) — **заголовок-эссе** (фраза, а не имя фильма).
  ✅ `Дебют Куросавы возвращается целым`
- Показ везде одинаковый: **«Русское»**, оригинал — вторичной строкой (приглушённо/латунь).
  Никогда не склеивать русское и оригинал в одно поле.

### 3. Подборки и скролл-истории (`layout: scrolly`)
Front-matter: `title`, `subtitle`, `kicker`, `cover`, `layout: scrolly`,
`kind: films|characters`, опционально `snap: page`, `media: stills`, `overline`.
- `kind: films` (нуар, молодёжь): запись — `<h3>` = **«Русское название»**,
  золотая строка-подпись = **оригинал**.
- `kind: characters` (женские персонажи): `<h3>` = **имя персонажа**,
  золотая строка-подпись = **«Русское название»**.
- Строка метаданных: `год · реж. Режиссёр · [страна | в гл. роли Актриса] · оценка · Letterboxd`.
- Обложка подборки (`cover`): **культовый кадр** из фильма (textless backdrop с TMDB),
  не постер с обрезанным текстом.
- **Первый слайд (стандарт):** круглое фото автора → «рассказывает **Карен Аванесян**»
  → рубрика **«Скролл-история»** (латунь, капитель). Поэтому в `kicker` рубрику
  НЕ дублируем — там только описание истории (или пусто).
- `overline` (например `Film Noir`) — латунный префикс В ОДНОЙ СТРОКЕ с названием
  (`Film Noir: Тёмная дюжина`), и на карточке, и на слайде; не отдельной строкой.
- История про один фильм: в front-matter можно дать `rating` и `letterboxd` —
  они покажутся на первом слайде (★ N/10 + иконка). Отдельный вводный «слайд-
  дубль» не делаем.
- Заметки-сцены: заголовок мысли — в `<h3>`, без служебных строк «Заметка N».

### 4. Letterboxd
- Ссылка строится как `https://letterboxd.com/tmdb/<tmdb_id>`.
- **Единый вид везде — пустой `<a class="lb">` (три фирменные точки фоном).**
  Никаких подписей «Letterboxd», стрелок ↗, рамок-кнопок и инлайн-SVG —
  иначе логотип задваивается (фон-точки + вставленный SVG).
  ✅ `<a class="lb" href="…" target="_blank" rel="noopener" aria-label="Letterboxd"></a>`
- В подборках/заметках — на строке метаданных. На странице рецензии — та же
  иконка, крупнее (через `.ext-links .lb`).

### 5. Постеры / кадры
- Постеры — TMDB, размер `w500` для карточек/сцен, `w780`/`w1280` для обложек.
  Локализуются сборкой. Без «киноплёнки» по краям.
- Язык постера — оригинальный язык фильма (см. правило в памяти проекта).

### 6. Визуальные токены (не хардкодить цвета для темизируемого)
- `--gold` — декор (звёзды, капитель-лейблы, линейки, «ИДЁТ»).
- `--accent` — только интерактив (ссылки, активный пункт, «Читать →»).
- Светлая тема: бумага `--bg`, чернила `--text`, лазурные шапка/футер,
  тёплые границы `--border`. Тёмная — кинематографичная, золото на тёмном.

## Чек-лист при добавлении контента
- [ ] `film` = только «Русское»; оригинал — в `original` (родное → латиница).
- [ ] `rating` — число 0–10; оценка показана общим компонентом, не вручную.
- [ ] Letterboxd — `letterboxd.com/tmdb/<id>`, иконка `.lb` на строке метаданных.
- [ ] Постер/обложка — TMDB, запекается сборкой; обложка подборки = кадр, не постер.
- [ ] Цвета берутся из токенов (`--gold`/`--accent`), не хардкодятся.
- [ ] `npm run build` проходит; коммит — только изменённые файлы.
