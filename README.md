# Караван идёт — авторский сайт о кино

Авторский сайт Карена Аванесяна: рецензии, лента с дневниками фестивалей, авторские подборки и публикации в прессе. Весь контент — простые текстовые файлы, добавлять новое легко.

## Структура

Разделы сайта по порядку: **Лента** (главная) · **Рецензии** · **Кинофестивали** · **Подборки** · **Публикации** · **Обо мне**.

```
cinema-reviews/
├── index.html                 ← шаблон-оболочка сайта (дизайн + клиентская логика)
├── build.mjs                  ← сборщик: пререндер каждой страницы в статический HTML
├── package.json               ← зависимости и команды (npm run build / dev)
├── about.md                   ← страница «Обо мне»
├── press.json                 ← раздел «Публикации» (тексты в других СМИ)
├── assets/                    ← картинки + скрипты
│   ├── render.js              ← общий модуль разметки (его делят браузер и сборщик)
│   ├── marked.min.js          ← локальная копия парсера Markdown (без CDN)
│   ├── logo.png               ← логотип канала  (положи сюда)
│   └── karen.jpg              ← твоё фото        (положи сюда)
├── feed/                      ← Лента (главная страница): заметки и мысли
│   ├── manifest.json
│   └── *.md
├── reviews/                   ← Рецензии
│   ├── manifest.json
│   └── *.md
├── festivals/                 ← Кинофестивали: репортажи и дневники
│   ├── manifest.json
│   └── *.md
├── collections/               ← Подборки (авторские коллекции)
│   ├── manifest.json
│   └── *.md
├── tools/
│   └── telegram_import.py     ← перенос постов из Telegram в Ленту
└── dist/                      ← результат сборки (генерируется, в git не хранится)
```

Контент по-прежнему добавляется правкой `.md` и `manifest.json` — как раньше.
Новое: перед публикацией запускается сборка (`npm run build`), которая превращает
каждую страницу в отдельный статический HTML с корректным заголовком, описанием и
карточкой для соцсетей. Благодаря этому рецензии индексируются поисковиками и красиво
разворачиваются ссылкой в Telegram. На Netlify сборка запускается автоматически.

Лента и Кинофестивали устроены одинаково (папка + `manifest.json` + `.md`-файлы с полями `title`, `date`, `tag`). Запись в фестивали кладётся в `festivals/`, обычная заметка — в `feed/`. По умолчанию скрипт импорта из Telegram складывает посты в Ленту (`feed/`).

## Сначала положи две картинки

В папку `assets/` добавь два файла **с такими именами**:

- `logo.png` — логотип канала «Караван идёт»
- `karen.jpg` — твоя фотография для страницы «О себе»

Без них сайт работает, просто покажет аккуратные заглушки.

## Как добавить рецензию

1. Создай `reviews/мой-фильм.md` (имя латиницей без пробелов — это адрес).
2. В начале — данные между двумя `---`:

```markdown
---
title: Однажды в Голливуде
year: 2019
director: Квентин Тарантино
country: США
rating: 4
poster: https://ссылка-на-постер.jpg
tags: драма, комедия
---

Текст рецензии. Можно **жирный**, *курсив*, ## заголовки и > цитаты.
```

3. Добавь фильм в `reviews/manifest.json` (так он появится на главной):

```json
{ "slug": "moy-film", "title": "Однажды в Голливуде", "year": "2019",
  "director": "Квентин Тарантино", "rating": "4", "poster": "", "excerpt": "Описание для карточки." }
```

`poster`/`cover` можно оставить пустыми (`""`) — будет заглушка с названием.

## Как добавить запись в Ленту (дневник с фестиваля)

Создай `feed/моя-запись.md`:

```markdown
---
title: Канны, день второй
date: 21 мая 2026
tag: Дневник · Канны 2026
---

Текст заметки.
```

И добавь строку в начало `feed/manifest.json` (свежие — сверху): `slug`, `title`, `date`, `tag`, `excerpt`.

## Как добавить подборку

Создай `collections/моя-подборка.md` с полями `title`, `subtitle`, `cover` и текстом, где можно ссылаться на свои рецензии так: `[Читать рецензию →](#/review/parasite)`. Затем добавь её в `collections/manifest.json` (`slug`, `title`, `cover`, `count`, `excerpt`).

## Как добавить публикацию в раздел «Пресса»

Просто допиши объект в `press.json`:

```json
{ "outlet": "Искусство кино", "title": "Заголовок статьи", "date": "Май 2026", "note": "колонка", "url": "https://ссылка-на-статью" }
```

Клик по карточке открывает оригинал статьи в новой вкладке.

## Перенос контента из Telegram-канала

Самый быстрый способ перенести старые посты в Ленту:

1. Установи **Telegram Desktop** (приложение для компьютера).
2. Открой свой канал → меню (⋮) → **«Экспорт истории чата»**.
3. Формат — **«Машиночитаемый JSON»**; отметь «Фото», если нужны картинки.
4. Telegram создаст папку с файлом `result.json`.
5. В терминале, в папке сайта, запусти:

```bash
python3 tools/telegram_import.py "/путь/к/ChatExport/result.json"
```

Скрипт создаст по записи в `feed/` для каждого поста и обновит `feed/manifest.json`. Запускать можно повторно — дубликаты он не создаёт. После импорта просмотри новые файлы и поправь заголовки: Telegram-посты часто стоит отредактировать под формат сайта.

## Черновики

Чтобы рецензия/подборка/заметка ещё не показывалась на сайте, добавь в её строку в
`manifest.json` поле `"draft": true`. Сборщик пропустит такую запись: ни карточки, ни
отдельной страницы, ни ссылки в карте сайта. Убери флаг — и она появится.

## Посмотреть сайт локально

Нужен Node.js (один раз поставь зависимости — `npm install`). Дальше одна команда:

```bash
npm run dev
```

Она соберёт сайт и поднимет сервер на `http://localhost:8000`. После правки контента
просто перезапусти команду (или `npm run build`, чтобы пересобрать без сервера).

Двойной клик по `index.html` по-прежнему **не сработает** — нужен именно локальный сервер.

## Где захостить (бесплатно)

1. **Netlify** (рекомендуется) — подключи репозиторий, и Netlify сам выполнит сборку.
   Настройки уже прописаны в `netlify.toml`: команда `npm run build`, публикуемая папка `dist`.
   Каждый `git push` пересобирает и публикует сайт.
2. **Vercel** — vercel.com, импорт репозитория; build command `npm run build`, output `dist`.
3. **GitHub Pages** — тоже подойдёт, но сборку нужно настроить через GitHub Actions
   (запускать `npm run build` и публиковать `dist`).

> Перетаскивание папки в Netlify Drop больше не подходит: сайту нужна сборка, которая
> пререндерит страницы. Собери локально (`npm run build`) и залей содержимое `dist/`,
> либо — проще — подключи репозиторий, чтобы хостинг собирал сам.

Итог остаётся статическим: быстрый, без сервера и базы данных. К любому хостингу можно подключить свой домен (например `karavan-idet.ru`), купив его отдельно (reg.ru, Namecheap — примерно 200–1500 ₽/год).

## Обновление: витрина лекций, связность, даты (июль 2026)

- **Главная**: карточки курсов в айдентике самих деков — три бокса в ряд первой секцией, манифест-пилюля с фото под hero, полоса «Пишу для» со счётчиками. Закреп «Выбор автора» по умолчанию выключен (вернуть: slug в `site.json → featured`); ряд «Кинофестивали» с главной убран (страница и пункт меню остались). Данные курсов — `tools/courses-data.mjs`, сводка попадает в `dist/site-data.json`.
- **Меню** (сайт и лендинг лекций): Рецензии · Лекции · Подборки · Кинофестивали · Публикации в СМИ · Заметки · Обо мне.
- **Шапка**: иконки Поиск · Telegram · RSS · Почта. Тумблер светлой темы временно закомментирован в `index.html` (светлая тема ждёт переработки).
- **Футер**: четыре колонки (о сайте · разделы · связь · «пишу для») + подпись «Собака лает, а караван идёт».
- **Рецензии**: дата публикации (`date` в `reviews/manifest.json`, ISO) и минуты чтения — строкой в конце текста (на карточках минуты остаются в мета); фестиваль показывается «печатью» без решётки; теги из frontmatter стали кликабельными метками со страницами `/tag/<slug>`; в конце — «мост» к лекции, где фильм в программе курса, бейдж подборки (если фильм упомянут в тексте подборки) и блок «Читать дальше» (близость по меткам/режиссёру/фестивалю). Обогащение считает `build.mjs` и кладёт в `dist/reviews/manifest.json`.
- **Поиск по сайту**: `/search` — клиентский, по рецензиям, подборкам, заметкам и СМИ.
- **RSS**: `pubDate` у записей, свежие сверху.
- **Картинки TMDB**: `npm run bake` скачивает все внешние постеры в `assets/img/` и переписывает исходники на локальные пути (запускать локально, где TMDB доступен; затем закоммитить). До запечки работает прежняя схема со скачиванием при сборке.
- **Лекции**: лендинг `/lectures/` перешит в дизайн-язык сайта (тёмная гамма, Playfair, карточные поверхности), айдентика каждого курса осталась цветной нитью его секции; файлы курса перенумерованы под структуру 12 лекций (`lecture-11-capitalism.html`, `lecture-12-politekonomiya.html`), внутри деков знаменатель «/ 12».
