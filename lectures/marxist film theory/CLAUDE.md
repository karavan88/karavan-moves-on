# CLAUDE.md — Марксистская теория кино

Авторский курс **Карена Аванесяна**. Десять самодостаточных HTML-лекций (слайд-деки) на русском для веб-сайта. Каждая лекция — один файл `lecture-NN-*.html`, разворачиваемый как отдельный маршрут (`/lecture-01` и т.д.). Аудитория — студенты социологии и кинотеории, уровень магистерского семинара.

Куратор и автор всех содержательных решений — Карен. Роль ассистента: технический сборщик + научный консультант. Перед исполнением — честная оценка соответствия (подходит ли фильм теме лекции), затем правка.

## Дизайн-система (конструктивизм; CSS идентичен во всех файлах)
- Палитра: `--ink #14110f` · `--oxblood #3a1714` · `--bone #ece6da` · `--red #d6342a` · `--ochre #c9962b`.
- Шрифты: Oswald (дисплей) / PT Serif (текст) / JetBrains Mono (утилитарный).
- Подпись: перфорированная киноплёнка-рейка слева = индикатор прогресса.
- Навигация: стрелки/пробел/клик по краям/свайп; счётчик; прогресс-бар; хеш в URL.
- На рейке каждого слайда: `Марксистская теория кино · авторский курс Карена Аванесяна`. На обложке — то же + название лекции.
- Слово «элективный»/«электив» НЕ должно встречаться нигде.

## Технические конвенции (обязательны при правках)
- **Динамическая нумерация.** Футеры используют `<span class="fpg"></span>` — JS заполняет их `NN — TOT`. Счётчик навигации: `<span class="cur" id="cur">01</span> / <span id="tot">00</span>`; JS ставит `#tot` из числа слайдов. НИКОГДА не вписывать статичные номера (`12 — 37`) — они ломаются при добавлении слайдов.
- **Каноничный `<script>`** (из lecture-10) обязателен: мульти-кадровый TMDB-загрузчик с `seen`-дедупликацией и `data-idx`, заполнение `.fpg`, установка `#tot`.
- **Структура модуля фильма = 5 слайдов:** intro · context · scene · form · reading. Два кадра на фильм: intro `data-idx="0"`, scene `data-idx="1"`. (Исключение: «Метрополис» в л.1 — 6 слайдов, 3 кадра.)
- **Футеры модуля:** intro `<span>Кейс NN · 1 / 5</span>`, scene `· 3 / 5`, reading `· 5 / 5`; context(2)/form(4) — `<span class="src">…</span>`. Всегда + `<span class="fpg"></span>`.
- **Каждая лекция заканчивается** слайдом «К просмотру» (фильмы лекции) И слайдом «К чтению».
- **Каждая лекция глубокая:** именованный теоретический аппарат (теоретики + работы + годы), честные «сила/предел» секции, кросс-ссылки на другие лекции, слайд актуальности.
- **Print-CSS** (`@media print`) уже во всех файлах: при «Сохранить как PDF» — все слайды по одному на альбомную страницу с тёмным фоном.
- Только обложка не имеет футера (поэтому `.fpg` на единицу меньше числа слайдов — это норма).

## TMDB (кадры)
- Вверху `<script>`: `const TMDB_API_KEY = ""`. Пусто → показывается дизайнерский плейсхолдер.
- Контейнер сборки НЕ имеет доступа к TMDB; ключ НЕ вставлять в коде от лица ассистента (риск безопасности) — это делает Карен у себя.
- Вставка ключа во все файлы разом (Mac): `sed -i '' 's/const TMDB_API_KEY = ""/const TMDB_API_KEY = "КЛЮЧ"/' lecture-*.html` (Linux — без пустых кавычек после `-i`).
- Загрузчик берёт `backdrops` (fallback `stills`). Постеры отдельно сейчас НЕ грузятся (не реализовано).

## Принципы отбора фильмов
- Фильм должен отвечать на КОНКРЕТНЫЙ теоретический вопрос лекции, а не просто относиться к теме.
- Интеллектуальная честность структурна: расхождения с марксизмом помечаются явно (Бодрийяр после «Зеркала производства» 1973 порвал с Марксом; Холл — «марксизм без гарантий»; риск «культурного популизма»).
- Карен предпочитает аргументированный pushback ДО исполнения, а не молчаливое согласие.

## Рабочий процесс правок
- Крупные блочные замены HTML: Python-сплайс по уникальным маркерам-комментариям (`<!-- ============ FILM N: … -->`); точечные — `str_replace`.
- После каждой правки — bash-валидация: число слайдов, число `.film` блоков, `data-tmdb`/`data-idx`, 0 «электив», 0 битых `</div></span>`, отсутствие статичных футеров, наличие `id="tot"`.
- Менять только: `<title>`, номер на рейке, узлы coursemap (cur/done), контент. CSS/JS — идентичны во всех файлах.
